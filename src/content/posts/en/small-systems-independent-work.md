Independent work creates repeated decisions: where to write, how to publish, where to track projects, how to explain current work, and how to make old material findable.

Small systems help by making those decisions boring.

## A site as a system

SignalPaper treats a site as a set of repeated patterns:

- Posts for arguments and notes
- Projects for applied work
- Tags for themes
- Now for current focus
- About for context
- Search for return visits

Each pattern is simple, but together they reduce maintenance friction.

## Fewer choices, better defaults

The best system is not the one with the most options. It is the one that lets you publish consistently without reopening the same decisions every week.

Start with the defaults. Remove what you do not use. Add structure only when the archive needs it.
