A good author site is not a profile page with extra links. It is a working archive that helps readers understand what you think, what you make, and where to go next.

The most important choice is hierarchy. A visitor should be able to answer three questions quickly:

- What is this site about?
- What should I read first?
- What else has this person built or published?

SignalPaper uses a small set of durable surfaces to answer those questions: writings, projects, tags, search, about, and now. Each one has a clear role. Writings carry arguments. Projects show applied work. Tags connect themes across time. Search helps readers return later.

## Do less, but make it legible

Author sites often become noisy because every new idea gets a new section. A stronger pattern is to keep the structure stable and let the archive grow inside it.

The site should not try to prove everything at once. It should make the best paths obvious.

## What to customize

Replace the demo content, update the author profile, and decide which features you actually need. If you do not use comments, analytics, or a newsletter, leave them off. A quiet default is easier to trust.
