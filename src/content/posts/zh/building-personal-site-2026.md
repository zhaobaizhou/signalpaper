个人网站变得更容易搭建，也更难有意义地维护了。以下是我思考那些真正重要的决策的方式。

## 真正重要的决策

### 1. 你会为它写作吗？

如果答案是否定的，技术问题就无关紧要了。

在你选择框架或设计布局之前，问问自己：你有写作习惯，还是你希望工具能创造一个？它不会。工具会让一个已有的习惯运作起来，或者成为一个漂亮的空城。

如果你还没有写作习惯，从简单的东西开始——哪怕是一个公开分享的腾讯文档——证明给自己看你会写。然后再建网站。

### 2. 网站实际上是为了什么？

个人网站对不同的人有不同的目的。明确你的：

- **写作存档**：网站是你思考的持久家园。你写，你发布，让它积累。
- **职业呈现**：网站展示你的工作，让访客容易理解你做什么。
- **项目枢纽**：网站记录并链接到你的项目，项目本身存在于别处。
- **组合**：大多数网站是某种组合，但你需要知道哪个功能是首要的。

目的决定结构。写作存档优先考虑可读性和可找到性。职业呈现优先考虑对正确访客的第一印象。

### 3. 你想让什么被搜索到？

SEO 首先不是一个技术问题。它是关于你想对什么词汇有排名，以及你是否真正在写这些内容。

任何现代框架的技术基础都很容易做到：语义化 HTML、适当的 meta 标签、合理的加载时间、移动端响应式。这些只需要一个下午。

困难的是：通过持续、实质性的写作在某个话题上建立权威。那才是难的部分，没有任何框架能帮你做到。

### 4. 你想拥有多少，委托多少？

这个范围是：

- **完全拥有**：自建服务器，自定义代码，完全控制权 + 完全维护负担
- **托管托付**：Vercel/Netlify 配合 Astro 或 Next.js 等框架——你拥有代码但把基础设施委托出去
- **托管建站器**：Squarespace、Webflow 等——你拥有内容但把其他一切都委托出去
- **平台**：微信公众号、小红书——你基本上什么都不拥有，但得到了它们的分发

拥有越多 = 控制权越多 + 维护成本越高。根据你想把时间花在基础设施还是内容上来选择。

对大多数想写作的人来说：带静态网站框架的托管托付是最优解。低维护，完全内容所有权，合理的定制空间。

## 不重要的事

- 你用哪个 JavaScript 框架（对内容网站来说都很好）
- 你用 CMS 还是 Markdown 文件（两者都有效）
- 设计是自定义的还是主题的（读者不在乎）
- 辅助色的确切色值

重要的是：你发布值得阅读的内容，在一个你会维护的网站上，有能正常工作的 URL。

其余的都是偏好。
