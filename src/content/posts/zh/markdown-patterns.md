这篇文章只做一件事：用一页内容把作者站最常见的 Markdown 场景跑一遍。

SignalPaper 强调安静、稳定的长文阅读，但真正决定体验的，往往是这些细节是否站得住：标题、链接、代码、引用、列表、表格和注释。

## 标题与行内文本

一篇文章首先要能把普通段落排好。它应该能自然地承载 **强调文字**、*斜体*、行内 `code`，以及像这样指向[内容模型文档](/zh/about)之外的链接而不显得凌乱。

当内容变长之后，层级也要足够清楚，让读者在细读之前先完成扫描。

## 列表

无序列表适合并列信息：

- 标题要短而清楚。
- 元信息应该帮助阅读，而不是抢注意力。
- 导航要存在，但不该过度打断正文。

有序列表适合步骤性内容：

1. 先把页面定义说清楚。
2. 再减少视觉摩擦。
3. 最后让归档更容易回头再看。

任务列表常见于公开构建日志或工作笔记：

- [x] 阅读进度
- [x] 相关文章
- [x] 静态搜索
- [ ] 你自己的内容和写作口径

## 引用与提示

> 一个好的发布主题，应该帮助读者保持方向感，但不让界面成为主角。

这条原则对文档页和文章页都成立。

## 代码

```ts
type DemoSection = {
  title: string;
  purpose: 'reading' | 'navigation' | 'documentation';
};

const sections: DemoSection[] = [
  { title: 'Markdown patterns', purpose: 'documentation' },
  { title: 'Long-form reading', purpose: 'reading' },
];
```

## 表格

| 元素 | 作用 |
| --- | --- |
| 标题 | 帮助读者快速扫描结构 |
| 列表 | 把密集内容拆成步骤 |
| 代码块 | 支撑技术写作 |
| 表格 | 紧凑地比较信息 |
