独立工作者几乎都会掉进一个陷阱。它的运作方式是这样的：你发现了一个新工具，它完美地解决了某一个问题，于是你开始想象它能不能解决所有问题。你把一切都迁移过去。六个月后，你再次启动这个循环。

我在这个循环里待了四年。然后我停了。

这篇文章不是一篇测评。它更像是一份关于"我最终为什么停止切换"的事后分析，以及现在这套稳定工具栈长什么样。

## 思路的转变

我用的工具不是市面上最好的工具。它们是我用得足够久、能够**在里面思考**的工具。这两者是有区别的。

当你足够了解一个工具——当界面消失、你只是在工作——你就获得了杠杆。这种杠杆是不能转移的。每一次迁移都在消耗它。

> 目标不是拥有最好的工具，而是拥有你已经忘记它存在的工具。

我现在评估工具只看一个维度：**三年后我还会用它吗？** 如果答案很可能是否，我就不开始。

## 写作：Obsidian + iA Writer

**Obsidian** 处理所有需要相互关联的内容：笔记、调研、项目追踪、决策日志。我把它当作第二大脑，不是写作应用。图谱视图对我基本没用。本地存储、纯 Markdown 格式，这两点不可或缺。

**iA Writer** 是我正式写作的地方。它只有一种模式：专注于文字。没有插件，没有图谱，没有数据库。打开快，然后消失。

交接很简单：当 Obsidian 里的某个想法准备好变成一篇真正的文章，它就进入 iA Writer。完成之后，进入网站的内容文件夹。

## 任务管理：纯文本

我试过 Things、Linear、Notion、Craft、Todoist。我现在用一个叫 `now.md` 的 Markdown 文件，存在 Obsidian 里。它只有三个部分：

```markdown
## 本周
- [ ] 写工具这篇文章
- [ ] 审阅 signalpaper 主题

## 本月
- ...

## 暂停区
- ...
```

这听起来太简单了，简单得不像是真的管用。但它确实管用。

原因是：为"收集"优化的任务管理工具，在"做决策"这件事上非常糟糕。当你有 200 个任务和无限的视图时，你把精力花在整理上，而不是执行上。

## 沟通：只用邮件

我关掉了 Slack。关掉了微信通知。每天只看两次邮件。

这是我做过的效率提升最大的一个改变。为了处理消息而不断切换上下文，每天会吃掉我 20-30% 的有效精力。

默认异步，改变了一切。

## 开发：终端 + VS Code + Cursor

没什么新奇的。大部分编辑用 VS Code，需要 AI 辅助时用 Cursor，其余全靠终端。`zsh` 配置极简。

我尝试过两次切换到 Neovim。最后决定，配置成本不值得。

## 我裁掉的工具

更有意思的清单是我移除的东西：

- **Notion**：太慢，太灵活，太容易建出精致的系统但从不使用它
- **Roam Research**：双向链接确实有用，但只能在浏览器里用、数据存在别人那里，这两点让我放弃了
- **任何习惯追踪 App**：当追踪一个习惯变成了习惯本身，你已经跑偏了

## 稳定工具栈

进入稳定清单的条件：必须本地存储（或导出为开放格式），价格便宜或免费，且我已经用了至少两年。

| 工具 | 类别 | 使用年限 |
|:---|:---|:---|
| Obsidian | 笔记 / 知识 | 3 年 |
| iA Writer | 长文写作 | 4 年 |
| VS Code | 代码编辑 | 5 年 |
| 终端 (zsh) | 系统操作 | 10 年+ |
| Fantastical | 日历 | 3 年 |
| Backblaze | 备份 | 5 年 |

就这六个。其余都在试用期。

## 真正的收获

工具不是瓶颈。你的注意力才是。

一旦接受这一点，问题就从"X 最好的工具是什么"变成了"我怎样减少管理工具的时间，多花时间真正使用它们？"

我的答案是：**停止切换**。
